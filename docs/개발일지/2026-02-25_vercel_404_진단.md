## 404 에러 원인 분석

### 핵심 원인: `app/`과 `src/app/` 이중 App Router 충돌

Next.js App Router는 `app/` 또는 `src/app/` 중 **하나만** 사용해야 한다. 현재 프로젝트는 두 곳 모두에 App Router 파일이 존재한다.

**로컬에서는 동작한 이유**: Next.js는 `app/`을 `src/app/`보다 우선 인식하고, `app/page.tsx`가 `src/app/page`를 re-export하므로 간접적으로 동작.

**Vercel에서 실패한 이유**:

1. **빌드 환경 차이**: Vercel의 빌드 환경에서 이중 `app/` 구조가 빌드 최적화 과정에서 충돌을 일으킬 수 있음
2. **layout.tsx 불일치**: `app/layout.tsx`는 `AuthProvider`를 포함하지만 `src/app/layout.tsx`는 미포함. 두 layout이 동시에 존재하면 Next.js가 혼동
3. **`src/app/(dashboard)/` 빈 디렉토리**: route group 디렉토리가 비어있으면 라우팅 트리 구성 시 오류 발생 가능
4. **proxy re-export 누락**: `frontend/features/`에 `auth/` 디렉토리가 없지만 `src/features/auth/`는 존재. `app/layout.tsx`가 `@/features/auth/AuthProvider`를 import하는데 `@/`는 `./src/*`를 가리키므로 경로 자체는 맞지만, 빌드 시 모듈 해석 순서에 따라 문제 가능

### 부가 원인: 환경변수

- `.env.local`이 `.gitignore`에 포함되어 Git에 push되지 않음
- Vercel 대시보드에서 환경변수를 별도로 설정했는지 확인 필요
- 특히 `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`가 없으면 Supabase 클라이언트 초기화 실패 가능
