# Ant Gravity Master Design System

## 1) 제품 디자인 방향
- 제품 유형: 웹뷰 기반 AI 주가 예측 SaaS
- 핵심 인상: 신뢰감 있는 금융 서비스 + 친근한 개미 캐릭터
- 톤 앤 매너: 깔끔함, 명확함, 약간의 유머, 과장 없는 자신감
- 디자인 목표:
  - 초보 투자자도 정보 우선순위를 빠르게 이해
  - 고급 사용자도 데이터 밀도에 불만이 없도록 구성
  - 캐릭터는 보조 내비게이터 역할만 수행하고 핵심 데이터 가독성을 방해하지 않음

## 2) 비주얼 컨셉
- 기본 스타일: Flat Design + Soft Rounded UI
- 레이아웃: 카드 중심 단일 컬럼(모바일) / 12컬럼 그리드(데스크톱)
- 시각 위계:
  - 1순위: 종목명, 현재 시그널(상승/중립/하락), 신뢰도
  - 2순위: 예측 근거, 주요 지표, 리스크 안내
  - 3순위: 개미 캐릭터 피드백, 코칭 문구
- 캐릭터 운영 원칙:
  - 상태형 캐릭터 3종: 기본, 경고, 축하
  - 모든 금융 판단은 숫자/지표와 함께만 노출
  - 캐릭터 단독으로 매수/매도 행동을 유도하지 않음

### 대비 기준
- 일반 텍스트 대비: 최소 4.5:1
- 큰 텍스트/굵은 텍스트 대비: 최소 3:1
- 상태 색상만으로 의미 전달 금지, 아이콘/문구 병행

## 4) 타이포그래피
- Heading: Plus Jakarta Sans 700/600
- Body: Plus Jakarta Sans 500/400
- Numeric: tabular-nums 활성화
- 본문 기본 크기:
  - Mobile: 16px / line-height 1.6
  - Desktop: 16-18px / line-height 1.6
- 숫자 데이터(가격/퍼센트): 15-17px, semibold

## 5) 간격/모서리/그림자
- Spacing scale: 4, 8, 12, 16, 20, 24, 32, 40
- Border radius:
  - 버튼/입력: 12px
  - 카드: 16px
  - 모달: 20px
- Shadow:
  - 기본 카드: `0 4px 16px rgba(15, 23, 42, 0.06)`
  - 강조 카드: `0 8px 24px rgba(37, 99, 235, 0.14)`

## 6) 컴포넌트 원칙
### 버튼
- 최소 터치영역 44x44px
- Primary/Secondary/Ghost 3종 고정
- 로딩 중 중복 클릭 방지

### 카드
- 데이터 카드와 캐릭터 카드 분리
- 캐릭터 카드는 전체 뷰 너비의 30%를 넘지 않음
- 핵심 지표 카드에는 항상 "기준 시각" 표시

### 차트
- 기본: 라인(시계열), 바(비교), 도넛(구성비)
- 차트 단독 해석 금지, 요약 문구와 수치 동시 노출
- 상승/하락 색상은 고정 규칙 유지(녹색/빨강 또는 프로젝트 표준)

### 피드백/에러
- 오류 상태 3단계:
  - 네트워크 오류: 재시도 버튼 제공
  - 데이터 지연: 마지막 업데이트 시각 표시
  - 권한 오류: 업그레이드 또는 재인증 가이드

## 7) 모션 원칙
- 전환 시간: 150ms-220ms
- transform/opacity만 사용
- 실시간 수치 갱신 애니메이션은 300ms 이내
- `prefers-reduced-motion` 활성 시 모션 최소화

## 8) 접근성/신뢰성 원칙
- 키보드 포커스 링 항상 표시
- 아이콘 전용 버튼은 aria-label 필수
- 폼 입력은 label 연결 필수
- 민감정보 저장 금지(GPS 등 개인정보 비저장)
- 투자 조언 문구 금지, 예측 근거/리스크 고지 필수

## 9) 캐릭터 디자인 가이드
- 개미 캐릭터 성격:
  - 친근하고 똑똑한 분석 도우미
  - 과장된 확신 표현 금지
  - 실패/하락 상황에서도 침착하고 중립적인 톤 유지
- 표정 시스템:
  - 기본: 미소 + 분석 보드
  - 경고: 차분한 표정 + 주의 아이콘
  - 축하: 절제된 긍정 반응
- 문구 예시:
  - "지금은 변동성이 커. 분할 관점으로 보자."
  - "신호는 긍정이지만 거래량 확인이 필요해."
  - "오늘은 관찰 우선 전략이 유리해 보여."

## 10) 랜딩/대시보드 구조
### 랜딩
1. 히어로: 가치 제안 + 신뢰 지표
2. 예측 근거 소개: 모델/지표 설명
3. 실제 화면 미리보기: 웹뷰 UX 강조
4. 요금/등급 안내: 무료/유료 차이 명확화
5. FAQ + 리스크 고지
6. CTA

### 대시보드
1. 상단: 종목 검색, 계정 상태, 마지막 갱신 시각
2. 핵심 지표 섹션: 예측 방향, 신뢰도, 변동성
3. 차트 섹션: 기간별 추이 + 이벤트 마커
4. 근거 섹션: 기술지표/감성/외부지표
5. 개미 코치 섹션: 행동 가이드(비강제)

## 11) 안티 패턴
- 캐릭터를 배경 장식처럼 크게 사용해 데이터 가독성 저하
- 의미 없는 글래스 효과 남용으로 대비 저하
- 상승/하락 상태를 색으로만 구분
- 긴 문장으로 결론을 숨기는 카피
- 모바일에서 차트/테이블 가로 스크롤 강제

## 12) 구현 체크리스트
- [ ] 아이콘은 SVG 세트로 통일
- [ ] 모든 클릭 요소 `cursor-pointer` 적용
- [ ] 포커스/호버 상태 시각 피드백 제공
- [ ] 모바일 375px에서 가로 스크롤 없음
- [ ] 데이터 카드에 기준 시각 표기
- [ ] 예측 결과에 근거 지표와 리스크 문구 동시 표시
- [ ] 등급별 기능 차이(기본/유료) UI에 명확히 표시
- [ ] `prefers-reduced-motion` 대응
- [ ] 텍스트 대비 4.5:1 이상
