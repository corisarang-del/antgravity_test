# 계약서: 문서 경로 동기화 계약

## 목적
문서 내 참조 경로를 현재 프로젝트 구조와 일치시키기 위한 입력/출력/검증 기준을 정의한다.

## 1. 입력 계약
- 입력 문서는 `design-system/`, `specs/`, `docs/` 범위에서 수집한다.
- 기준 구조는 현재 저장소 루트 실제 디렉터리 기준으로 판단한다.
- `frontend/app` 및 `frontend/src/app` 병행 구조를 모두 인지해야 한다.

## 2. 출력 계약
동기화 완료 산출물은 아래를 충족해야 한다.
- 깨진 경로가 0건이어야 한다.
- 구버전/중복 경로는 canonical 경로로 치환돼야 한다.
- 품질 게이트 실행 위치와 명령이 문서에 명시돼야 한다.
- 기록 경로(`docs/개발일지/`, `docs/prompt/`)가 최신 구조와 일치해야 한다.

## 3. 검증 계약
- 구조 정합성: 경로 참조 일치율 100%
- 깨진 경로/중복 경로/구버전 경로: 0건
- 온보딩 동선: 핵심 문서 5개 이상, 목표 15분 이내
- 품질 게이트: `pnpm lint`, `pnpm typecheck` 명시 및 실행 결과 기록

## 4. 예외 계약
아래 항목은 예외 허용하되 근거 기록이 필요하다.
- 구조 마이그레이션 진행 중인 임시 경로
- 외부 의존으로 접근 불가한 참조 문서

## 5. 완료 정의
- 입력/출력/검증 계약 전부 충족
- 검토 체크리스트 통과
- 변경 이력(`docs/개발일지/`, `docs/prompt/`) 기록 완료
