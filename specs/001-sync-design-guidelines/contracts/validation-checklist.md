# 품질 검증 체크리스트

## 문서 정합성
- [ ] 기준 소스(frontend)와 문서 정의가 충돌하지 않는다.
- [ ] 공통 규칙과 페이지 오버라이드의 역할이 분리되어 있다.
- [ ] 금지 패턴이 명시되어 있다.
- [ ] 용어가 문서 전체에서 통일되어 있다.

## 웹뷰/접근성
- [ ] safe-area 규칙이 문서와 코드에 모두 존재한다.
- [ ] 터치 타깃 최소 44x44 규칙이 명시되어 있다.
- [ ] 포커스 가시성 규칙이 명시되어 있다.
- [ ] 라이트/다크 규칙이 모두 존재한다.
- [ ] 본문 텍스트 대비 4.5:1 미만 항목이 0건이다.

## 헌법 준수
- [ ] GPS/정밀 위치 저장 금지 원칙 위반 항목이 없다.
- [ ] 컴포넌트 PascalCase/변수 camelCase 원칙을 유지한다.
- [ ] 품질게이트(`pnpm lint`, `pnpm typecheck`) 결과가 기록되어 있다.
- [ ] 개발일지/프롬프트 기록 파일이 갱신되어 있다.
